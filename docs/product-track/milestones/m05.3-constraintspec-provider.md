## Progress (2026-06-22)

The constraint-plane spec/snapshot/decision model is codified, and the decision
enum is produced for the first time.

- `crates/causlane-core/src/domain/constraint_spec.rs`: `ConstraintKind`
  (`Freeze` / `TokenBudget{limit}` / `Restrict{note}`), `ConstraintSpec`,
  `ConstraintSnapshot { snapshot_id, epoch, constraints, active_leases }`, and the
  pure `resolve_constraints(snapshot, claims, scopes)` arbiter — the first producer
  of `ConstraintDecision` (Allow / Wait / Deny / AllowWithRestrictions). Adds
  token-budget arbitration (previously absent — only exclusive conflicts were
  checked). Reuses `claim_modes_conflict`, `ResourceClaim`/`ClaimMode`/`LeaseRef`/
  `ConstraintEpoch` and the existing decision/blocker/violation types.
- `crates/causlane-core/src/contract.rs`: new `ConstraintProvider` kernel trait;
  `KernelContracts` implements it by delegating to `resolve_constraints`
  (re-exported from `lib.rs`). The hexagonal `ConstraintProviderPort`
  (`application/ports.rs`) supplies snapshots.
- **Executable checks (no prose-only):** per-decision unit tests (freeze→Deny,
  over-budget→Deny, token-full→Wait, active exclusive conflict→Wait,
  restrict→AllowWithRestrictions, clear→Allow), a **fail-closed precedence**
  property (Deny > Wait > Restrict > Allow), token-budget edges (`==limit` allowed),
  and a `ConstraintKind` exhaustiveness guard.

**Risk/limitation:** `ConstraintKind` is intentionally minimal (Freeze /
TokenBudget / Restrict, no guessed taxonomy); rate-limit and other kinds, and
wiring snapshots into the replay/barrier path (`constraint_snapshot_id`), are
follow-ups (M05.4/M05.7). Pure additive `causlane-core` (builds standalone; no
reducer/verify-path change). Deeper Alloy/P/Lean modeling of the arbiter is S10
follow-up; formal contour untouched.
