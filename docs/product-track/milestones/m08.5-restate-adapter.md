# M08.5 — Restate adapter

- **Stage:** S08 — Runtime shell и adapters
- **Status:** `done_or_near_done`
- **Purpose:** Durable handler/workflow adapter; optional feature.

## Delivery note

M08.5 adds the optional `causlane-runtime/restate` feature. The adapter wraps an
`ExecutorService` as a Restate handler/workflow bridge over a JSON wrapper around
opaque host payload bytes. A host-owned decoder turns the opaque payload into
the existing `GuardedExecutionJob`.

The authority order is unchanged and remains centralized in `GuardedExecutor`:

```text
Restate Json<opaque payload> -> host decoder -> GuardedExecutionJob
  -> GuardedExecutionRequest -> ExecutorService::call
```

Restate context, workflow state, invocation metadata and handler wrappers are
non-authoritative. They cannot supply authz evidence, alter policy identity,
mint capabilities, or bypass spend-time capability admission. The default error
mapper turns decode and guarded execution failures into Restate terminal errors.

## Limitation

This milestone is a handler/workflow bridge, not a durable Causlane job schema.
It does not make `causlane-core` domain types serde-serializable and does not
define a stable Redis/HTTP/Kafka Restate payload format. Hosts own payload
compatibility until a later schema milestone explicitly defines it.
