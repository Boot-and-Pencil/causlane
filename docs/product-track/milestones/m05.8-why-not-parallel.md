## Progress (2026-06-22)

why-not-parallel — the machine-readable blocker/rationale that closes the S05 exit gate
("Dispatcher может объяснить `ready`, `blocked`, `why-not-parallel`") — is codified as an
**aggregator** that wraps the already typed causes of M05.2–M05.7 and re-derives nothing.

- `crates/causlane-core/src/domain/why_not_parallel.rs`:
  - `enum NotParallelReason` — one vocabulary; every variant wraps an existing cause by value:
    `Frontier(FrontierBlock)` (M05.5), `ConstraintWait(ConstraintBlocker)` /
    `ConstraintDeny(ConstraintViolation)` (M05.3/M05.7), `LaneRejected(LaneRejection)` (M05.2),
    `DrainRegion{target, scope}` (M05.6), `BlockedOnFact{fact}` /
    `BlockedOnActiveScope{scope, held_by}` (M05.4 index).
  - `why_not_parallel(op_id, &WhyNotParallelInputs)` — unions the caller-supplied decider
    outputs in a fixed order (deterministic for replay); `Allow` / `AllowWithRestrictions`
    contribute nothing. `WhyNotParallel::is_parallelizable()` ⟺ no cause aggregated.
  - `reason_from_frontier` (lift one rejection); `pair_conflict` (pairwise — only the TD-010
    shared-write-scope bar; lane budget is per-lane, surfaced per-op).
  - Named carriers `DrainedWriteScope` / `ActiveScopeHolder` (no positional tuples).
- **Executable checks (no prose-only):** the load-bearing property
  `aggregation_is_faithful_and_sound` asserts, across the product of all input categories,
  **value-faithfulness** (every supplied cause appears as exactly one reason carrying its
  values — nothing invented or dropped) and **soundness** (`is_parallelizable()` ⟺ zero
  causes — a rejected op is never reported runnable, a clear op never blocked), with a
  non-vacuity guard exercising all seven reason variants; plus unit tests for the frontier
  lift, lane surfacing, allow-contributes-nothing, and pairwise conflict. ADR-0005 updated
  (Codification M05.8). The upstream causes carry their own negative controls (M05.5 frontier
  rejection tests, M05.3 fail-closed precedence, I-007/I-010 receipts).

**Risk/limitation:** "machine-readable" here is the typed Rust enum (the kernel is Rust-first);
`causlane-core` has no `serde`, so a JSON / `--json` surface is a downstream concern (the
replay/CLI crates, like `ReplayExplain`) — out of scope. The aggregator consumes *outputs* of
the existing deciders and adds no scheduling logic, so it cannot disagree with them. Pure
additive `causlane-core` (no formal-bound type / codegen change → no bundle-hash or formal-IR
regeneration). With M05.8 every S05 milestone (M05.1–M05.8) is `done_or_near_done` and the S05
exit gate is met — the stage is ready to close out.
