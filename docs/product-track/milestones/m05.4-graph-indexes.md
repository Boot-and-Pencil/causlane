## Progress (2026-06-22)

The graph-index layer that feeds frontier selection (M05.5) is codified.

- `crates/causlane-core/src/domain/graph_index.rs`: `OpId` (action + op index),
  `GraphNode` (lane, required `FactKind`s, written `Scope`s), and `GraphIndex`
  holding four deterministic (`BTreeMap`/`BTreeSet`) indexes — `wait_by_fact`,
  `wait_by_scope`, `active_by_write_scope`, `ready_by_lane`. Classification:
  "structurally ready" ⟺ all required facts produced **and** no active op writes a
  conflicting scope; constraint-plane (M05.3) and lane-capacity (M05.2) budgets are
  layered on top by frontier selection, not here.
- **Incremental maintenance:** `add_node` / `mark_produced` / `mark_active` /
  `mark_complete` reclassify only the affected nodes through a single `recompute`
  primitive; `from_state` is the full rebuild; `rebuilt()` rebuilds from the
  index's own base state. Queries: `waiting_on_fact` / `waiting_on_scope` /
  `active_writers` / `ready_in_lane`.
- **Executable checks (no prose-only):** the load-bearing property
  `incremental_matches_full_rebuild` asserts `index == index.rebuilt()` after every
  event across a graph exercising all four indexes (fact dependency, write-scope
  conflict, multiple lanes), with a non-vacuity guard that each index was
  populated; unit tests for fact-unblock, active-writer scope-block/unblock, and
  `from_state` determinism.
- `Scope` / `LaneId` / `ActionId` / `FactKind` gained `PartialOrd, Ord` (additive)
  to serve as deterministic `BTreeMap` keys. ADR-0005 updated (Codification M05.4).

**Risk/limitation:** structural readiness only (facts + write-scope conflicts);
constraint/lane/resource budgets are M05.5. The incremental recompute finds
affected nodes by scanning `nodes` for the changed key (correct; reverse-index
optimization deferred). Pure additive `causlane-core` (builds standalone; no
reducer/verify-path change). Deeper Alloy/P/Lean modeling of the index invariant is
an S10 follow-up; the formal contour is untouched.
