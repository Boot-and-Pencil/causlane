## Progress (2026-06-22)

Runtime constraint updates are codified **on top of** the existing I-010
truth-preservation authority (`constraint_update.rs` /
`ConstraintUpdate::preserves_committed_truth`), without duplicating or altering it ‚Äî so
I-010's Kani/Verus/P receipts stand untouched.

- `crates/causlane-core/src/domain/constraint_runtime.rs`:
  - **four update kinds:** `RuntimeUpdateKind { Capacity{lane, capacity}, Quota{resource,
    scope, limit}, Freeze{scope}, RateLimit{resource, scope, max_per_epoch} }`.
  - **with epoch:** `next_epoch` (saturating `+1`), `epoch_advances` (strict monotonicity);
    `apply_to_snapshot` produces a new `ConstraintSnapshot` at the next epoch ‚Äî `Quota`
    upserts the `TokenBudget`, `Freeze` adds the freeze idempotently, `Capacity`/`RateLimit`
    leave snapshot constraints unchanged; stale-epoch leases are dropped (`lease_current` ‚Äî a
    lease is valid only within its grant epoch).
  - **future-only:** `RuntimeUpdate { kind, applied_at }` + `governs(admission_epoch)` holds
    iff `admission_epoch >= applied_at` (same semantics as a drain, consistent with I-010).
  - **no truth rewrite (I-010 reuse):** `truth_rewrite_of(kind)` returns an empty rewrite
    mask, so a runtime update preserves committed truth for *any* committed state.
  - **rebuild frontier (M05.4/M05.5 reuse):** `apply_capacity(lanes, kind)` yields the updated
    lane registry that `select_frontier` is re-run against.
- **Executable checks (no prose-only):** the load-bearing property
  `apply_preserves_truth_bumps_epoch_and_drops_stale_leases` asserts, across all four kinds,
  strict epoch advance + committed-truth preservation (over every committed state) +
  stale-lease drop, with a non-vacuity guard that constraint-resident kinds change the
  constraints while lane-side/rate-limit kinds do not; an integration test shows a capacity
  tightening rebuild a smaller frontier; unit tests for epoch monotonicity, `lease_current`,
  future-only `governs`, freeze idempotency, and quota upsert. ADR-0005 updated (Codification
  M05.7). Negative control for I-010 itself
  (`constraint_update_cannot_rewrite_committed_truth_nondet`, Kani) already exists.

**Risk/limitation:** `RateLimit` **enforcement** is a follow-up ‚Äî there is no constraint-plane
primitive for it yet (M05.3 deferred it), so the kind exists and bumps the epoch (sharing the
update/epoch/rebuild path) but is not wired into `resolve_constraints`. Unfreeze is a follow-up
(Freeze is add-only, matching the drain‚Üífreeze direction). Surfacing the blocking reason is
M05.8. Pure additive `causlane-core` (no change to `CommittedTruth` / `ConstraintUpdate` /
`preserves_committed_truth` / `AuditEventKind` / codegen ‚Üí no bundle-hash or formal-IR
regeneration). Deeper Lean/Verus modeling of the epoch-bump theorem (L-013) is an S10
follow-up; the formal contour is untouched.
