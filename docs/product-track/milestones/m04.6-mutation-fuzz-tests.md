## Progress (2026-06-22)

`crates/causlane-replay/tests/mutation_fuzz.rs` — a programmatic mutation + fuzz
harness over the replay oracle, public API only, no new dependency/fixture.

- **Targeted mutations (15):** each rewrites the valid `release_promote_success`
  trace and asserts `verify_with_bundle` rejects with one exact `ReplayErrorCode`
  — I-001 (ExecutionWithoutBarrier, MissingBarrierPayload, BadImpactSetHash,
  CapabilityMissing, CapabilityMismatch), I-003 (ProjectionWithoutAnchor,
  AnchorNotObservedTruth, AnchorAttestationMismatch), I-006 (ConflictingLeases),
  I-007 (DrainFenceWithActiveOverlap), I-008 (EventAfterClosed), I-009
  (RequiredWitnessMissing, WitnessSelectorMismatch, WitnessBindingMismatch) and a
  structural PlanHashMismatch. (Authz I-009 codes + ObservedWithoutExecution stay
  covered by the `*_invalid` suite, the `src/tests.rs` `verify_events` slice tests
  and the M04.5 contract harness — this bundle runs authz disabled.)
- **Totality / fuzz:** a deterministic corpus (truncated "small worlds" + per-event
  drop / duplicate / retype-to-each-of-17-kinds / null-payload / corrupt-hash +
  predicate/binding edits) asserts `verify_with_bundle` always RETURNS `Ok|Err` —
  never panics/hangs/aborts. Anti-vacuity: `total/2 < rejects < total` (most cases
  reject, a benign minority verify — the oracle discriminates). The oracle is
  panic-free **by construction** (`#![forbid(unsafe_code)]`, no unwrap/expect/
  indexing on the verify path — audited), so no `catch_unwind` is needed.
- A compile-time `event_kind_table_is_complete` guard fails the build if a new
  kernel event kind is added until the fuzz retype table is updated.

Design was produced by a multi-agent design sweep (panic surface + mutation
catalog + fuzz approach) and adversarially reviewed; two spec mispredictions
(`MissingRequiredBarrier`→`ExecutionWithoutBarrier`, `MissingBarrierImpactSet`→
`BadImpactSetHash`) were corrected against the real oracle. Tests-only; no
protocol-critical source or formal-contour change.
