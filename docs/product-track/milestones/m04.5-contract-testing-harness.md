## Progress (2026-06-22)

A data-driven YAML contract-testing harness over the scenario corpus.

- `causlane contract test --manifest <.yaml> [--json]`
  (`crates/causlane-cli/src/cli_contract.rs`): for each suite `{registry,
  scenarios}` in the manifest, compile each scenario against the bundle
  (`compile_scenario`, M04.3) and replay it (`verify_explain`, M04.4), asserting
  the scenario's OWN declared contract — `expected_replay_result` plus
  `expected_error_code` for negatives. Reports per-case `expected`/`actual`/`ok`
  (human or `--json`); **fails closed** — any mismatch (or manifest/IO error)
  exits non-zero.
- Manifest `contracts/contract-tests.yaml`: 25 cases across 5 suites
  (release_promote, authz, projection_readonly, multi_action, merge). A new
  scenario gets contract coverage by adding one line — no Rust. Generalizes the
  ad-hoc bash `authz_verify` loop in `tools/formal-verify-all` into a typed,
  reusable runner.
- Gate: `just contract-test` runs the manifest (25/25 pass).
- Tests (`cli_contract::tests`): the contract-satisfaction logic accepts a
  matching outcome and rejects a wrong code / wrong outcome (non-vacuity); an
  end-to-end mis-pairing (authz scenario under the non-authz registry) is
  reported as a failing case with a non-zero exit.

Pure CLI-boundary orchestration over existing functions — no new protocol
semantics; formal contour untouched. **Follow-ups:** a `dispatch_test!` macro for
inline Rust contract cases, and adapter-certification suites once adapters land
(S08); wiring `formal-verify-all` onto `contract test` (needs the full toolchain).
