## Progress (2026-06-22)

The tier model is codified as a pure kernel module.

- `crates/causlane-core/src/domain/tier.rs`: ordered `Tier` enum (`Admission →
  Planning → Dispatch → Barrier → Execution → Observation → Projection →
  Closure`; derived `Ord` is the authority order) with `Tier::ALL`,
  `next_tier()` and `authority()` (a stable machine-readable token per tier).
- `reached_tier(LifecycleStage) -> Option<Tier>` grounds the tiers in the S03
  lifecycle reducer (`New` has reached no tier; the other eight stages map 1:1).
  No parallel source of truth.
- **Executable check (no prose-only):** `tier_is_monotonic_under_valid_lifecycle_transitions`
  exhaustively iterates the finite `stage(9) × event(17) × profile(6)` space and
  proves every valid `reduce_lifecycle` transition keeps the reached tier
  non-decreasing (an action never drops to a lower authority), with a
  non-vacuity guard that strict advances occur. Plus unit tests (stage mapping,
  ordered pipeline, distinct authorities) and a `Tier::ALL` exhaustiveness guard.
- ADR-0005 updated with the codification note.

Pure, additive `causlane-core` module derived from the already formally-covered
lifecycle reducer (it builds standalone; no reducer/verify-path change). Deeper
standalone Alloy/P/Lean modeling of the tier model is S10/M02.x hardening
follow-up (lean/lake unavailable on this host); the formal contour is untouched.
