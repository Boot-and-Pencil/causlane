# M08.7 — Adapter certification

- **Stage:** S08 — Runtime shell и adapters
- **Status:** `done_or_near_done`
- **Purpose:** Bounded certification matrix for existing adapters; retry/cancel/truth orchestration deferred.

## Delivery note

M08.7 adds a bounded adapter certification matrix for the runtime adapters that
exist today. Execution-bearing adapters are certified by shared Rust tests that
drive Apalis and Restate through the same `GuardedExecutor` contract:

- authorized jobs reach the executor exactly once;
- missing authz never reaches the executor;
- expired/refused capability never reaches the executor;
- adapter envelope or metadata cannot create authz evidence;
- produced refs survive the adapter wrapper.

The machine-readable evidence ledger is
`docs/product-track/adapter-certification-matrix.json`. It also records audit
append-only and tracing non-authority evidence from existing adapter tests.

## Limitation

This is not a new retry, cancellation, or durable truth-commit implementation.
Those semantics remain deferred to reliability/formal-depth milestones where
worker retry, cancellation/supersession, and durable observed-truth commit
become first-class runtime protocols.
