## Progress (2026-06-22)

Two pure, fail-closed, additive deciders in `causlane-core` plus a thin runtime
composition close the read side of S06. Additive — the new types are absent from
`kani_target.rs`; no codegen / formal-IR / bundle-hash regen.

- `crates/causlane-core/src/domain/projection_authz.rs` (new): read/projection
  authorization at the `may_project` stage. `read_authz_gate(decisions, req)`
  **reuses** the ADR-0011 `authz_gate` verbatim (no new verdict ladder → no
  precedence drift), pinned to `MAY_PROJECT_STAGE`, after restricting decisions to
  `req.actor`. `authz_gate` does not bind the requesting actor, so the wrapper's
  actor filter is what makes a read actor-scoped: a `may_project` `Allow` minted for
  another reader denies `Missing`. `ProjectionReadRequest<'a>` bundles the coordinates
  (borrow struct, avoids `too_many_arguments`).
- `crates/causlane-core/src/domain/redaction.rs` (new): the masking **mechanism**, no
  taxonomy. `RedactionPolicy { revealable: BTreeSet<FieldPath> }` is a fail-closed
  **allowlist** — `classify_field` / `apply_redaction` reveal a field iff it is listed
  `revealable`, else redact, so a sensitive/unknown/forgotten path is masked, never
  leaked. `RedactionView { revealed, redacted }` partitions a projection's paths.
  `FieldPath` is an exact-match token (no normalization; a host obligation).
- `crates/causlane-runtime/src/projection_guard.rs` (new): `guard_projection_read`
  authorizes the read (`read_authz_gate`) and only on `Allowed` emits the
  `RedactionView`; a denial returns `ProjectionReadError::Unauthorized { stage, reason }`
  and no view — so no field reaches an unauthorized reader (mirrors `GuardedExecutor`).
- `docs/07-security-and-authz.md`: the `may_project` read-authz gate (actor-bound,
  deny-by-default) and the fail-closed redaction mechanism.
- **Executable checks (no prose-only):** the redaction property test
  `apply_redaction_matches_oracle` exhausts the field universe × every `revealable`
  subset against an independent set-algebra oracle, with non-vacuity and
  order-independence; read-authz negative controls (`read_for_wrong_actor_is_denied`,
  `read_wrong_stage_does_not_authorize`, missing/wrong-binding/expired/explicit-deny);
  and the runtime guard's negative controls (`read_without_may_project_allow_is_denied`,
  `read_for_wrong_actor_is_denied`, `authorized_read_yields_redaction_view`).

**Risk/limitation:** **additive only — the formal contour is untouched.** Redaction
completeness and path **canonicalization** are host obligations: the kernel masks by
exact-match set membership; the S07/M07.5 class/profile layer compiles into this
same allowlist and does not replace it. **Out of scope:** value-byte masking / JSON
shaping (host/adapter);
a typed `projection_policy` in plan-hash material (would touch the formal contour;
read-authz rides the existing `may_project` authz channel instead); self-read / SoD /
step-up on reads (a host issues a stricter `may_project` policy via the existing
`AuthzPolicy`). Replay-trace wiring of `read_authz_gate` is deferred-additive (M06.4
precedent); the runtime `guard_projection_read` is the live enforcement point. No serde.
