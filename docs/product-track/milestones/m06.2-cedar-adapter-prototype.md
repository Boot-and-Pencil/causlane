## Progress (2026-06-22)

The Cedar adapter prototype is codified as a **mapping-only** module — it proves the
Causlane authz vocabulary maps faithfully onto Cedar's Principal/Action/Resource/Context
(PARC) shape, embeds **no** `cedar-policy` engine, and makes **no** authorization
decision (the deny-by-default gate stays the sole authority).

- `crates/causlane-runtime/src/adapters/cedar.rs` (NOT `causlane-core` — engine-free
  core; runtime hosts adapters, depends only on `causlane-core`):
  - Cedar-shaped, Causlane-owned types: `CedarEntityType { Subject, Action, Predicate }`
    (a closed schema-label enum, not a free string), `CedarEntityRef`, `CedarPrincipal` /
    `CedarAction` / `CedarResource`, `CedarContext { circumstance_ref, plan_hash, stage }`,
    `CedarRequest`, and `CedarAuthzRequest<'a> { call: &ActionCall, plan, stage }` (it
    **reuses** `ActionCall`, redefining no authz term).
  - `to_cedar(req) -> CedarRequest` — the locked, grounded mapping:

    | Causlane | source | Cedar PARC slot |
    |---|---|---|
    | subject (`ActionCall.subject_ref`, "the subject the action acts on") | `&str` | **Principal** `Subject::"<ref>"` |
    | action verb (`ActionCall.action_id`) | `ActionId` | **Action** `Action::"<id>"` |
    | predicate (`ActionCall.predicate`, "the typed shape an action operates over") | `PredicateId` | **Resource** `Predicate::"<id>"` |
    | circumstance + plan hash + stage | `&str`/`PlanHash`/`&str` | **Context** `{ circumstance_ref, plan_hash, stage }` |

- **Executable checks (no prose-only):** exact-slot faithfulness unit; a load-bearing
  property `every_coordinate_is_faithfully_carried` (flipping any one of the six
  coordinates changes the mapped request — totality + observability, no dropped field,
  non-vacuity over each); and `same_value_in_different_inputs_lands_in_distinct_slots`
  (the same literal in two inputs lands in two distinct typed PARC slots — no aliasing).

**Risk/limitation:** **mapping-only** — real `cedar-policy` policy evaluation is a deferred,
feature-gated follow-up (the prototype proves the schema maps; it does not run Cedar). The
adapter is **decision-neutral**: it implements no `AuthzEvaluator`, so it cannot become a
second authority — the deny-by-default gate (`authz_gate`, ADR-0011 / I-009) remains the
sole authority. Engine-free core untouched: **no new dependency** (`Cargo.lock` unchanged),
no `causlane-core` change, no codegen / bundle-hash / formal-IR change — the formal contour
is untouched. `Casbin` / `AuthZEN` / `OpenFGA` sketches are M06.3 ("do not bake one engine
into core"); approvals/step-up/capability/redaction are M06.4–M06.7.
