# M08.3 — Executor port/adapters

- **Stage:** S08 — Runtime shell и adapters
- **Status:** `done_or_near_done`
- **Purpose:** Tower-like Service port; hard effects only with capability.

## Delivery note

M08.3 adds a dependency-free service-shaped executor seam in `causlane-runtime`.
It does not replace `causlane_core::ExecutorPort`; that port remains the
low-level semantic executor contract and still requires `&ExecutionCapability`.

The runtime-level service shape is:

- `GuardedExecutionRequest<'a>`: borrows the barrier, authz evidence, policy,
  spend time and op;
- `ExecutionOutcome`: returns produced refs;
- `ExecutorService`: `call(request)` executes one guarded request.

`GuardedExecutor<E>` implements `ExecutorService` by reusing the same internal
helper as legacy `spend_barrier`. The authority order stays centralized and
single-sourced:

```text
authz -> derive capability -> spend_admits -> ExecutorPort::execute
```

`adapters::executor` now exposes two small executor adapters:

- `NoopExecutor` for tests/examples that need no produced refs;
- `FunctionExecutor<F>` to adapt a closure into `ExecutorPort`.

## Authority boundary

Executor adapters do not decide authorization or capability validity.
`GuardedExecutor` is the supported hard-effect seam; raw `ExecutorPort`
implementations are lower-level and must only be called with a valid capability
from trusted composition code.

## Failure mode

`ExecutorService::call` returns `SpendError`: unauthorized barriers fail before
capability derivation, invalid/expired capabilities fail before executor entry,
and inner executor failures are wrapped as `SpendError::Execute`.

## Tests and controls

Runtime tests cover service success, unauthorized refusal, expired-capability
refusal, legacy/service equivalence, `FunctionExecutor` receiving the exact
op/capability pair, and `NoopExecutor` returning no refs under a derived
capability.
