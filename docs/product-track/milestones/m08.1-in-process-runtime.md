# M08.1 — In-process runtime

- **Stage:** S08 — Runtime shell и adapters
- **Status:** `done_or_near_done`
- **Purpose:** Tokio partition loops, bounded queues, semaphores for capacity, no global lock.

## Delivery note

M08.1 adds the optional `causlane-runtime/tokio-runtime` feature. The default
runtime build remains dependency-minimal; Tokio is pulled only when this feature
is enabled.

The feature exposes `causlane_runtime::in_process`:

- `InProcessRuntime`;
- `InProcessRuntimeConfig`;
- `InProcessRuntimeError`;
- `InProcessRuntimeEvent`;
- `InProcessEffectHandler`.

The runtime is constructed with a fixed partition set. Each partition owns its
local queue and dependency/completion state. Submitters route each task with an
explicit `PartitionKey`; unknown partitions are rejected. Queues are bounded by
`partition_queue_bound`: `submit` waits for ingress capacity, while `try_submit`
returns `QueueFull` when the ingress queue or admitted pending set has no room.

Effect execution is capped by one runtime-wide Tokio semaphore
(`max_concurrent_effects`). There is no global queue lock: the only global
coordination is immutable partition sender lookup, ticket sequencing, event
broadcast, and the configured capacity semaphore.

## Authority boundary

The in-process runtime does not create semantic authority. It uses
`validate_host_task` for the stable host seam and executes only through the
host-supplied `InProcessEffectHandler`. Authorization, hard-effect capability
spending, durable audit truth, retry policy, cancellation, secret resolution,
and product idempotency remain external adapter/host responsibilities.

## Failure mode

This is an ephemeral in-process runtime, not a durable queue. Handler failure
emits `InProcessRuntimeEvent::Failed`, does not mark the task complete, and
leaves dependents blocked on their missing dependency. Failed tasks are not
retried by M08.1.

## Tests and controls

Runtime unit tests cover host API rejection, partition-local idempotency
suppression, bounded queue refusal, independent partition concurrency, semaphore
capacity, dependency order, and handler failure blocking dependents.
