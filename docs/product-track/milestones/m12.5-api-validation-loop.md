# M12.5 — API validation loop

- **Stage:** S12 — Beta integrations и ecosystem validation
- **Status:** `planned`
- **Purpose:** Closed loop over realistic synthetic examples, property/fuzz testing and performance scale testing before API freeze.

## Required outputs

- normative doc/spec update;
- machine-readable artifact where applicable;
- tests/replay/formal receipt where applicable;
- coverage/readiness status update where applicable;
- risk/limitation note if not fully enforced;

## Seeded inventory

`docs/product-track/api-validation-loop-plan.json` now records the selected
candidate surfaces for the validation loop and binds each surface to existing
synthetic example, property/fuzz and performance-scale evidence where that
evidence exists. All selected surfaces now have terminal
`accepted_for_freeze` classifications backed by synthetic, property/fuzz and
performance-scale evidence.

`examples/facade-kernel-ergonomics` adds the first facade-only synthetic example
for `public_facade_and_core_kernel`: it depends only on `causlane` and exercises
admission, barrier policy and frontier selection through curated facade imports.

`examples/facade-kernel-operator-workflow` adds a near-real facade-only operator
workflow for the same surface: it checks admission, barrier/truth policy,
frontier selection, constraint decisions, lease validation and fail-closed
controls through public `causlane` APIs.

`verification/fuzz/fuzz_targets/facade_kernel_frontier.rs` adds the facade/kernel
property/fuzz target for the same surface: it byte-drives public facade and
curated core APIs without adding a second semantic authority.

`docs/formal/impact/2026-06-29-m12-5-facade-fuzz-long-run.md` records the
15-minute dispatcher long-run for `facade_kernel_frontier`: 5,105,061 executions
over 901 seconds with status 0 and no crash/reproducer artifacts.

`docs/formal/impact/2026-06-29-m12-5-facade-performance-scale.md` records the
dispatcher Criterion scale run for the facade/kernel surface: frontier selection
and exclusive lease grant are measured without introducing a latency threshold.

`docs/formal/impact/2026-06-29-m12-5-facade-api-feedback-classification.md`
records the facade/kernel API feedback decision: the current public facade and
curated core/kernel APIs are accepted for freeze from this evidence slice, with
no Rust API, manifest or generated-readiness changes.

`examples/replay-diagnostics` adds a replay/explain synthetic example for
`replay_scenario_explain`: it checks accepted, invariant-rejected and structural
bundle-hash rejected release-promotion traces through `verify_explain`, including
human and JSON diagnostics.

`examples/replay-operator-diagnostics` adds a near-real replay diagnostics
workflow for the same surface: it checks release execution, read-only sidecar
projection, multi-action histories, structural bundle provenance and strict
bundle-binding controls through public `causlane-replay` and
`causlane-contracts` APIs.

`docs/formal/impact/2026-06-29-m12-5-replay-fuzz-long-run.md` records 15-minute
dispatcher long-runs for `replay_trace_json` and `replay_scenario_yaml`:
63,502,840 and 7,182,729 executions respectively, both with status 0 and no
crash/reproducer artifacts.

`docs/formal/impact/2026-06-29-m12-5-replay-performance-scale.md` records the
dispatcher Criterion scale run for the replay surface: bundle-bound replay
verification and human explain rendering are measured without introducing a
latency threshold.

`docs/formal/impact/2026-06-29-m12-5-replay-api-feedback-classification.md`
records the replay surface API feedback decision: the current public replay
scenario, trace and explain APIs are accepted for freeze from this evidence
slice, with no Rust API, manifest or generated-readiness changes.

`examples/contracts-boundary-ergonomics` adds a contracts/bundle/plan-hash
synthetic example for `contracts_registry_bundle_plan_hash`: it checks registry
compilation, bundle artifact reload, template resolution, plan hashing and
impact hashing through public `causlane-contracts` APIs.

`examples/contracts-registry-bundle-workflow` adds a near-real multi-predicate
contracts workflow for the same surface: it checks registry validation,
compiled-bundle reload, template resolution, plan-template cache lookup, plan
hashing and impact hashing through public `causlane-contracts` APIs, with
negative controls for tampered bundle artifacts, missing template bindings,
blank required authz policy ids and mutated plan material.

`docs/formal/impact/2026-06-29-m12-5-contracts-fuzz-long-run.md` records the
first 15-minute dispatcher long-run for `registry_yaml_compile`: 7,749,966
executions over 901 seconds with status 0 and no crash/reproducer artifacts.

`docs/formal/impact/2026-06-29-m12-5-contracts-performance-scale.md` records the
dispatcher Criterion scale run for the contracts surface: registry
normalization, release-promotion plan hashing and compiled-bundle JSON load are
measured without introducing a latency threshold.

`docs/formal/impact/2026-06-29-m12-5-contracts-api-feedback-classification.md`
records the contracts surface API feedback decision: the current public
contracts registry, bundle and plan-hash APIs are accepted for freeze from this
evidence slice, with no Rust API, manifest or generated-readiness changes.

`examples/runtime-guarded-audit-projection` adds a runtime authz/audit/projection
synthetic example for `runtime_dispatch_audit_projection`: it checks
`GuardedExecutor`, `InMemoryAuditLog`, `TraceProjectingAuditLog` and guarded
projection redaction through public runtime APIs, with negative controls for
missing execution authz, expired capability spend, missing `may_project` authz
and duplicate audit ids.

`examples/runtime-operator-workflow` adds a near-real multi-operation runtime
host workflow for the same surface: it executes a rollout-style barrier through
public guarded execution APIs, appends the resulting audit trace, projects trace
spans and checks guarded dashboard projection redaction plus independent
negative controls for execution authz, capability expiry, projection authz,
actor scoping and duplicate audit ids.

`verification/fuzz/fuzz_targets/runtime_guarded_audit_projection.rs` adds the first
property/fuzz seed for `runtime_dispatch_audit_projection`: it byte-drives
public guarded execution, audit append, trace projection and projection redaction
APIs without adding a second semantic authority. A terminal runtime API feedback
classification now consumes this fuzz lane.

`docs/formal/impact/2026-06-29-m12-5-runtime-fuzz-long-run.md` records the first
15-minute dispatcher long-run for that runtime fuzz target: 10,502,706 executions
over 901 seconds with status 0 and no crash/reproducer artifacts. A terminal
runtime API feedback classification now consumes this fuzz lane.

`docs/formal/impact/2026-06-29-m12-5-runtime-performance-scale.md` records the
first dispatcher Criterion scale run for the same runtime surface:
`runtime_guarded_audit_projection_flow` measures 512 guarded operations, 1,028
audit events and one projection read with a 1.4399 ms mean. The surface uses no
latency threshold, but its API feedback is classified alongside the synthetic
and fuzz lanes.

`docs/formal/impact/2026-06-29-m12-5-runtime-api-feedback-classification.md`
records the runtime surface API feedback decision: the current public runtime
composition APIs are accepted for freeze from this evidence slice, with no Rust
API, manifest or generated-readiness changes.

The M12.5 loop now has terminal `accepted_for_freeze` classifications for all
