## Seeded inventory

`docs/product-track/api-validation-loop-plan.json` now records the selected
candidate surfaces for the validation loop and binds each surface to existing
synthetic example, property/fuzz and performance-scale evidence where that
evidence already exists. All selected surfaces remain classified as `pending`;
the seeded inventory is not terminal validation evidence.

`examples/facade-kernel-ergonomics` adds the first facade-only synthetic example
for `public_facade_and_core_kernel`: it depends only on `causlane` and exercises
admission, barrier policy and frontier selection through curated facade imports.

`examples/replay-diagnostics` adds a replay/explain synthetic example for
`replay_scenario_explain`: it checks accepted, invariant-rejected and structural
bundle-hash rejected release-promotion traces through `verify_explain`, including
human and JSON diagnostics.

`examples/contracts-boundary-ergonomics` adds a contracts/bundle/plan-hash
synthetic example for `contracts_registry_bundle_plan_hash`: it checks registry
compilation, bundle artifact reload, template resolution, plan hashing and
impact hashing through public `causlane-contracts` APIs.

`examples/runtime-guarded-audit-projection` adds a runtime authz/audit/projection
synthetic example for `runtime_dispatch_audit_projection`: it checks
`GuardedExecutor`, `InMemoryAuditLog`, `TraceProjectingAuditLog` and guarded
projection redaction through public runtime APIs, with negative controls for
missing execution authz, expired capability spend, missing `may_project` authz
and duplicate audit ids.

`fuzz/fuzz_targets/runtime_guarded_audit_projection.rs` adds the first
property/fuzz seed for `runtime_dispatch_audit_projection`: it byte-drives
public guarded execution, audit append, trace projection and projection redaction
APIs without adding a second semantic authority. The selected surface remains
`pending` until fuzz findings, API feedback and performance-scale findings are
classified.

`docs/formal/impact/2026-06-29-m12-5-runtime-fuzz-long-run.md` records the first
15-minute dispatcher long-run for that runtime fuzz target: 10,502,706 executions
over 901 seconds with status 0 and no crash/reproducer artifacts. The surface
still remains `pending` until API feedback and performance-scale findings are
classified.

`docs/formal/impact/2026-06-29-m12-5-runtime-performance-scale.md` records the
first dispatcher Criterion scale run for the same runtime surface:
`runtime_guarded_audit_projection_flow` measures 512 guarded operations, 1,028
audit events and one projection read with a 1.4399 ms mean. The surface still
remains `pending` until API feedback is classified across the synthetic, fuzz and
performance lanes.
