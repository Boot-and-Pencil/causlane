# M07.6 — Support bundle

- **Stage:** S07 — Observability, explainability и DX
- **Status:** `done_or_near_done`
- **Purpose:** Sanitized bundle with trace, graph slice, route rationale, environment/tool report.

## Delivery note

M07.6 adds `causlane support-bundle build`, which writes a sanitized JSON support
artifact from an existing compiled bundle, replay trace and graph snapshot. The
artifact includes bundle metadata, replay explain output, the M07.2 graph export
model, a formal-doctor environment/tool report and the M07.5 support-bundle
redaction report.

The implementation deliberately composes existing authorities. Replay diagnosis
uses `ReplayTrace::verify_explain`; graph context uses the graph export DTO; and
sanitization uses `RedactionSurface::SupportBundle` plus the class-to-allowlist
compiler from M07.5. There is no support-bundle-only denylist or second masking
engine.

## Limitation

Support bundle v1 does not embed raw trace documents. Subject/circumstance
binding values, raw authorization payloads, execution-capability payloads and
keyed attestations are omitted or represented as redacted summaries. Hosts still
own any value-byte shaping outside this typed diagnostic artifact.
