## Evidence

- `EffectSignature` (`crates/causlane-core/src/domain/effect.rs`):
  reads/writes/produces/requires/invalidates/conflict_domains/hardness. It is a
  data signature; its one protocol-relevant predicate is
  `is_mutable()` = (has writes) ∨ (Hard impact) — the rule deciding whether an op
  mutates state and therefore needs a barrier/lease.
- Tests: `is_mutable_is_writes_or_hard` (exhaustive over writes×hardness) and
  `projection_only_is_empty_and_not_mutable`. The `conflict_domains`/hardness fields
  feed conflict/drain reasoning (I-006/I-007) which is enforced and Kani-covered at
  the `LeaseTable`/`DrainFenceCheck` layer (see M03.5), not in this data type.
