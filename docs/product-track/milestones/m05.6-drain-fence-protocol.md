## Progress (2026-06-22)

The drain/fence protocol is codified **on top of** the existing I-007 fence-acquisition
authority (`drain.rs` / `DrainSemantics::can_acquire_fence`), without duplicating or
altering it — so I-007's Kani/Alloy/P receipts stand untouched.

- `crates/causlane-core/src/domain/drain_protocol.rs`:
  - **domain/global targets:** `DrainTarget { Domain(Scope), Global }` + `covers` (reuses
    `ScopeOverlap`).
  - **safe points:** `at_safe_point(target, leases, now, contracts)` — the `Domain` case
    **delegates** to `can_acquire_fence` (one rule, no divergence); the `Global` case is that
    rule generalized to every scope (no lease active and non-expired anywhere).
  - **disjoint domains:** `drains_independent(a, b, oracle)` — two drains may proceed in
    parallel iff their targets are disjoint; a `Global` drain is never independent.
  - **frozen sidecars:** `op_admissible_during_drain(effect, target, oracle)` — a read-only
    sidecar (`!is_mutable()`) stays admissible (frozen); a mutable op is blocked only if it
    writes into the drained region.
  - **drain epochs:** `DrainRequest { target, epoch }` reuses `ConstraintEpoch`; `governs`
    holds only from its own epoch onward (future-only, consistent with I-010).
- **Executable checks (no prose-only):** the load-bearing property
  `safe_point_agrees_with_the_i007_authority` exhausts the lease active/expired/overlap space
  asserting `at_safe_point` agrees with the I-007 authority (+ global generalization), with a
  non-vacuity guard over both outcomes; unit tests for covers, independence (disjoint vs
  global), frozen-sidecar admissibility, and epoch governance. ADR-0005 updated (Codification
  M05.6). Negative controls for I-007 itself (`drain_with_active_lease_invalid`,
  `lease_during_drain_invalid`) already exist in the replay suite.

**Risk/limitation:** this is the pure protocol layer. Applying a drain as a runtime `Freeze`
constraint and rebuilding the frontier on the epoch bump is **M05.7**; surfacing the blocking
reason (why-not-parallel) is **M05.8**. Pure additive `causlane-core` (no change to
`DrainFenceCheck` / `fence_acquirable` / `can_acquire_fence` / `AuditEventKind` / codegen → no
bundle-hash or formal-IR regeneration). Deeper Lean/Verus modeling of the drain-epoch protocol
is an S10 follow-up; the formal contour is untouched.
