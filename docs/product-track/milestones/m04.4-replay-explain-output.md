## Progress (2026-06-22)

Replay now produces a structured explanation of any outcome.

- `ReplayError::invariant()` (`crates/causlane-replay/src/error.rs`) maps every
  error variant to the protocol invariant it violates (I-001..I-009), or `None`
  for structural/loading/provenance errors.
- `crates/causlane-replay/src/explain.rs`: `CausalLocation` (per-variant
  extraction of action/event/anchor/barrier/witness/requirement/scope/plan/stage)
  and `ReplayExplain` (accepted, invariant, error_code, error_detail,
  causal_location, checked_invariants, bundle/trace hash) with `to_json_pretty()`
  / `to_human()`. `ReplayTrace::verify_explain()` reuses the `verify_verdict`
  path.
- CLI: `causlane replay verify --bundle … --trace … --explain [--json]`
  (`crates/causlane-cli/src/cli_replay.rs`) prints the diagnosis and exits
  non-zero iff the trace is rejected.
- Tests: `crates/causlane-replay/tests/explain.rs` — accepted (no
  invariant/location, empty location omitted from JSON), I-001 located by action,
  I-006 located by scope. Diagnoses reuse the 38 stable replay error codes, so
  the negative-control scenario suite already exercises the explain mapping.

No new predicate or protocol semantics: `explain` is a read-only projection of the
existing verify path, so the formal contour and receipts are untouched.
