## Progress (2026-06-22)

One command compiles a scenario into all three artifacts.

- `causlane scenario compile --scenario <.yaml> --bundle <.json> --out-dir <dir>
  [--kernel-secret <secret>]` emits, from a single source:
  - `<id>.trace.json` — bundle-bound (P0-005), optionally attested (P1-006);
  - `<id>.formal-ir.json` — the target-neutral Formal IR ("formal facts"), which
    already carries `scenario_hash` + `expected_result`/`expected_error_code`;
  - `<id>.expectation.json` — a typed `ScenarioExpectation` manifest of the
    declared replay outcome.
- `crates/causlane-cli/src/cli_scenario.rs`: pure `compile_scenario()` (no I/O,
  unit-tested) returning `CompiledScenario`, plus the `compile_scenario_to_dir()`
  writer. Reuses the existing projections — `ReplayScenario::to_trace_bound`,
  `mint_capability_attestations`, `scenario_facts::alloy_scenario_facts_from_yaml`
  and `CodegenContracts::build_ir` — so the compiled artifacts are byte-identical
  to the separate `scenario emit-trace` / `formal ir emit` commands.
- Tests (`cli_scenario::tests`): a success scenario compiles to a bundle-bound
  trace that strictly verifies + a `pass` expectation; a negative scenario carries
  `expected_result=fail` + `expected_error_code`, and its emitted trace actually
  fails replay as declared.

Pure CLI-boundary orchestration over existing functions — no new protocol
semantics, formal contour and receipts untouched. **Follow-up:** rewire
`scripts/check-verification-full.sh` to drive `scenario compile` instead of the separate
emit steps (deferred because the full gate needs the absent Alloy/Lean toolchain
to re-run end-to-end here).
