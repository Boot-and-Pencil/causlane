# M07.5 — Redaction policy

- **Stage:** S07 — Observability, explainability и DX
- **Status:** `done_or_near_done`
- **Purpose:** Audit/log/projection/replay/support-bundle redaction classes.

## Delivery note

M07.5 adds the typed redaction-class layer in `causlane-core`. Hosts classify
known field paths as `Public`, `Operational`, `Restricted`, or `Secret` inside a
`SurfaceRedactionProfile` tagged for one of the supported observability/support
surfaces: audit, log, projection, replay, or support bundle.

The profile compiler is deliberately thin: `compile_redaction_policy` reduces a
class profile to the existing M06.7 `RedactionPolicy` allowlist. `apply_redaction`
remains the only masking mechanism, so classification and application cannot
drift. The class allowlist is fail-closed: the default reveals no classes, and the
provided `public_only()` helper reveals only fields explicitly classified
`Public`.

Duplicate declarations for the same path are also fail-closed. A path is compiled
as revealable only if every declaration's class is revealable, so an accidental
weaker duplicate cannot override a stronger classification.

## Limitation

M07.5 defines classification and policy compilation only. It does not perform
value-byte masking or JSON shaping; hosts/adapters must enumerate canonical field
paths and apply the returned `RedactionView` to concrete payloads.
