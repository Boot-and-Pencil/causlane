## Progress (2026-06-22)

The owned authorization-policy entity + default-deny decider is codified **on top of**
the existing deny-by-default gate (`authz.rs`: `classify_authz_decision` / deny-wins
`authz_gate`, ADR-0011 / I-009), without duplicating or altering it.

- `crates/causlane-core/src/domain/authz_policy.rs`:
  - `AuthzPolicyId` (typed id) + `AuthzPolicyModel { id, version, stages: Vec<String>,
    freshness_max_age }` — the core analogue of the `causlane-contracts`
    `AuthzPolicyManifest` DTO. `stages` are the **lifecycle-stage tokens** the gate and
    `AuthzDecisionRef.stage` already use (e.g. `execution_barrier_logged`), not a new
    vocabulary. `expected()` projects the borrowed `AuthzPolicy` view (single source of
    the id/version + freshness bound).
  - `decide_authz_policy(policy, decisions, stage, action, plan, predicate_id, now) ->
    AuthzPolicyOutcome { Allowed, StageNotAuthorized, DecisionDenied(AuthzDenyReason) }`:
    fail-closed in two ordered steps — policy gate (`admits_stage`, **precedence**),
    then **delegate** to `authz_gate` (binding, deny-wins, P0-010, freshness/expiry are
    re-derived nowhere → no drift from the live gate or replay).
- **Executable checks (no prose-only):** the load-bearing property
  `policy_model_is_fail_closed_with_gate_precedence` asserts over a grid that an
  unauthorized stage denies before classification (independent of the decisions), an
  authorized stage mirrors `authz_gate` exactly, `Allowed` ⟹ authorized ∧ gate-allowed,
  and a decision reason is reported only for an authorized stage — with non-vacuity over
  all three outcomes and temporal boundary cases; plus unit tests (real-lifecycle-stage
  allow, precedence over a would-be allow, deny-wins delegation, missing→`Missing`).
  ADR-0011 updated (Codification M06.1). The decision-level negative controls
  (no-policy-deny, expired/stale/forward-dated, `PolicyMismatch`) already exist in
  `authz.rs` and are inherited via delegation.

**Risk/limitation:** **engine-agnostic** — no `RBAC`/`ABAC`/`ReBAC`, no engine (`Cedar` is
M06.2; `Casbin`/`AuthZEN`/`OpenFGA` are M06.3). "Machine-readable" is the typed Rust enum;
the `AuthzPolicyManifest → AuthzPolicyModel` JSON lowering will live in `causlane-contracts`
(single source) — documented, deferred. Approval-as-action (M06.4), step-up/SoD (M06.5),
capability enforcement (M06.6), and projection redaction (M06.7) are later S06 milestones.
Pure additive `causlane-core` (no formal-bound type / codegen change → no bundle-hash or
formal-IR regeneration); the formal contour is untouched.
