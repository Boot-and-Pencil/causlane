# M08.4 — Apalis adapter

- **Stage:** S08 — Runtime shell и adapters
- **Status:** `done_or_near_done`
- **Purpose:** Rust-native jobs backend; certification tests.

## Delivery note

M08.4 adds the optional `causlane-runtime/apalis` feature. The adapter wraps an
`ExecutorService` as an Apalis/Tower service over an owned `GuardedExecutionJob`
carried in Apalis's request envelope.

The authority order is unchanged and remains centralized in `GuardedExecutor`:

```text
Apalis Request<GuardedExecutionJob> -> GuardedExecutionRequest -> ExecutorService::call
```

Apalis worker metadata, context and extensions are non-authoritative. They cannot
supply authz evidence, alter policy identity, mint capabilities, or bypass
spend-time capability admission.

## Limitation

This milestone is a service bridge, not a durable payload schema. It does not
make `causlane-core` domain types serde-serializable and does not define a
Redis/Postgres/SQLite Apalis storage wire format.
