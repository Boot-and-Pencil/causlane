## Progress (2026-06-22)

Approval is codified as a typed, auditable action bound to the exact target it
authorizes — the three-coordinate binding `action_id + plan_hash + impact_set_hash`.

- `crates/causlane-core/src/domain/approval.rs` (new, pure):
  - `ApprovalVerb { Approve, Deny }` — the gate verb abstraction, mirroring the
    `gate.approved` / `gate.denied` events (`AuditEventKind::GateApproved` /
    `GateDenied`).
  - `ApprovalRef { approval_event_id, verdict, action_id, plan_hash, impact_set_hash,
    actor }` — a recorded approval; reuses existing id/hash types. The impact binding
    is **mandatory** (a deliberate strict tightening of the *optional*
    `WitnessBinding.impact_set_hash`): an approval with no impact binding is not
    expressible and authorizes nothing.
  - `classify_approval(...) -> Result<(), ApprovalDenyReason>` checks coordinates
    action → plan → impact **before** the verb (so a misbound `gate.approved`
    authorizes nothing and a misbound `gate.denied` reports the drifted coordinate,
    not `ExplicitDeny`); `approval_gate(approvals, action, plan, impact_set) ->
    ApprovalOutcome { Approved, Denied(reason) }` is fail-closed and **accumulate-then-
    decide** so the outcome is order-independent. Precedence: `ExplicitDeny` (deny-wins)
    ≻ exact `Approve` ≻ `WrongAction` ≻ `WrongPlan` ≻ `WrongImpactSet` ≻ `Missing`.
- **Executable checks (no prose-only):** the load-bearing property
  `approval_gate_matches_oracle_and_is_order_independent` exhausts the
  {target,other}³-coordinate × {Approve,Deny} grid (singles + all pairs), asserting
  the gate matches an independent reference oracle **and** is order-independent
  (`gate(xs) == gate(reverse(xs))`), with non-vacuity over every outcome; plus unit
  controls (`exact_approve_authorizes`, `deny_wins_over_approve`,
  `misbound_deny_is_wrong_impact_not_explicit_deny`, empty→`Missing`).

**Risk/limitation:** **additive only — the formal contour is untouched.** The
formal-bound `AuthzDecisionRef` is deliberately **not** modified (it binds
action+plan+predicate and is on the live-gate ⇄ replay parity path fed to codegen;
adding `impact_set_hash` there would force a bundle-hash/formal-IR regen). The decider
is **pure and grounds nothing** — it trusts each `ApprovalRef` was faithfully extracted
from a grounded producer event; grounding (event exists, kind matches verb, attests the
triple) and wiring `ApprovalRef` into `AuditEvent`/runtime/replay are later (additive,
non-breaking). Step-up/MFA/SoD + approval freshness (`issued_at`/`expires_at`) are M06.5;
capability spend at the barrier is M06.6; projection redaction is M06.7. No serde.
