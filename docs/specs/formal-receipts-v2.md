# Formal receipt schema v2

Receipt v2 separates the kind of receipt from the tool result and binds the
generated artifact back to source material.

Required codegen receipt fields:

```text
schema_version = 2
receipt_kind
artifact_kind
target
tool
tool_version
generator_version
source_bundle_hash
formal_ir_hash
scenario_hash
core_model_hash
generated_artifact_hash
command
expected_result
actual_result
invariant_ids
scope
checked_at
```

`formal stale-check` rejects receipts whose `schema_version` is not `2`, whose
`source_bundle_hash` differs from the compiled bundle, whose `formal_ir_hash`,
target or scenario hash differs from the generated header, or whose artifact
hash differs from the generated bytes.

Expected-fail controls must record their actual result; they are not success
receipts for the checked property.
