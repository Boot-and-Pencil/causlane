# Canonical Serialization v1

Canonical Serialization v1 is compact UTF-8 JSON produced from concrete Rust
`Serialize` structs with deterministic field order.

Hash-critical material:

```text
BundleHash
PlanHash
ImpactSetHash
ScenarioHash
GeneratedArtifactHash
```

Rules:

- Hash material must use concrete structs, not dynamic JSON maps.
- Field order is the Rust struct declaration order.
- Set-like data must be stored as sorted vectors or otherwise deterministic
  ordered containers before hashing.
- Hashes are rendered as `sha256:<lowercase hex>`.
- Bundle artifacts hash `BundleBody`; the outer `bundle_hash` is excluded.
- Scenario hashes are computed from the typed scenario struct, not raw YAML
  bytes.
- Generated artifact hashes are over generated bytes, including hash headers.

The golden test is `canonical_serialization_v1_is_compact_and_ordered` in
`causlane-contracts`.
