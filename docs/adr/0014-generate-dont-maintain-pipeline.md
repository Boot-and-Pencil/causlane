# ADR-0014: Formal models are generated from the compiled bundle

- Status: accepted
- Date: 2026-06-05
- Supersedes: -
- Superseded by: -

## Context

A concrete, bundle-specific formal artifact that is hand-maintained becomes a
second specification. It drifts from the contract that runtime, replay, and
tests actually consume, and drift is not mechanically detectable. ADR-0006
already requires that formal models be tied to the compiled dispatch bundle;
this ADR operationalizes that requirement so the tie is enforced, not aspirational.

## Decision

Concrete, bundle-specific formal artifacts MUST be GENERATED from the
`CompiledDispatchBundle`, never hand-maintained as a parallel specification.

The pipeline is:

```text
registry.yaml + JSON schemas + scenario catalog
  -> causlane bundle compile
  -> compiled_dispatch_bundle.json + bundle_hash
  -> causlane formal generate (alloy|p|kani|verus)
  -> generated artifacts + receipts
```

Crate and CLI surface:

```text
causlane-codegen : alloy / p / kani / verus generators.
causlane-cli     : bundle compile, bundle validate,
                   formal generate, formal stale-check, replay verify.
```

Generated layout:

```text
verification/formal-full/alloy/generated/*.als
verification/formal-full/p/generated/*.p
verification/formal-full/kani/generated/*.rs
verification/formal-full/verus/generated/*.rs
verification/formal-full/receipts/*.json
```

Every generated file MUST carry a header:

```text
// GENERATED BY causlane / source_bundle_hash: sha256:... /
// generator_version: ... / do not edit manually
```

A receipt MUST record `tool`, `bundle_hash`, `generated_artifact_hash`,
`command`, `status`, `checked_at`, and `scope`.

The existing hand-written `verification/formal-full/alloy/causlane_core.als` and
`verification/formal-full/p/DispatcherProtocol.p` remain GENERIC EXPLORATORY sketches only. They
are NOT authoritative and MUST NOT be extended into bundle-specific facts.

READINESS GATE: real Alloy/P/Kani/Verus modeling work is DEFERRED until the
contract surface is stable. The gate is met when bundle v0.1 compiles,
`plan_hash` is computed, the event model has witnesses/anchors/leases, the
replay JSON loader exists and fails closed on I-001/I-002/I-003/I-008, authz
denies by default, `MergeProtocol::None` is specified, the barrier binds
witness + lease + `impact_set_hash`, a codegen path exists for at least Alloy,
and the coverage matrix lists generated input. See
`docs/11-contract-hardening-plan.md`.

## Consequences

Easier:

```text
the formal layer attacks the SAME contract that runtime, replay, and tests
  consume;
drift between specification and formal artifacts is mechanically detectable.
```

Harder:

```text
a codegen toolchain and deterministic bundle hashing MUST exist before any
  formal modeling work begins.
```

## Alternatives considered

Hand-maintaining bundle-specific models in parallel with the contract was
rejected: it creates a second source of truth, invites silent drift, and cannot
be checked mechanically — exactly the failure ADR-0006 warns against.

## Enforcement

- Docs: `docs/05-formal-modeling-strategy.md` records the deferral and the
  readiness gate; `docs/11-contract-hardening-plan.md` defines the gate criteria.
- Formal: `verification/formal-full/README.md` marks the current hand-written files as exploratory
  and non-authoritative; generated artifacts MUST carry the GENERATED header.
- Replay: `causlane replay verify` checks generated artifacts against the
  bundle they claim as source.
- Runtime: generators consume the compiled bundle and `bundle_hash` (ADR-0009);
  no bundle-specific fact may originate outside the bundle.
- Tests: CI runs `causlane formal stale-check` — if the bundle changed, generated
  artifacts are STALE and their receipts are invalid; manual edits to generated
  files are rejected by a hash check; the coverage matrix MUST list generated
  input per invariant.

Relates to ADR-0006 (formal modeling stack) and ADR-0009 (bundle_hash).
