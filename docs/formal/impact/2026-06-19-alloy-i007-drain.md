## Summary

The generic Alloy core models I-001/002/003/008 (`Enforced`) + I-006 (lease
conflict) + I-009 (binding assertions); the previous increment added I-008. This
extends Alloy to model **I-007 (a `drain.fence_acquired` over a scope must have no
active — granted, not-yet-released — overlapping EXCLUSIVE lease on that scope)** —
the *structural* core of the drain rule. Alloy has no notion of time, so the
expiry refinement (a drain over an *expired* lease) stays the replay oracle's job,
exactly like authz temporal facts. This makes Alloy a 4th independent lane for
I-007's structural core (replay/P/Kani already cover it).

- New module `alloy_drain.rs` (split from `alloy.rs` for the 800-line cap) projects
  each `DrainFenceAcquired` event whose scope is backed by a declared `LeaseScope`
  atom into a `DrainFence { dfEvent, dfScope }` binding, reusing the lease's
  `scope_sig()` so the drain scope and lease scope unify to the SAME atom (the crux
  that makes the overlap fire).
- The `DrainFence` sig, `GeneratedDrainFacts` fact, the `GeneratedDrainFenceClear`
  assertion and its `check` are emitted **for every scenario**. Drain-free scenarios
  pin `DrainFence = none` so the assertion holds vacuously (and cannot invent
  phantom drain atoms that would falsely fail the positive). `obligations.rs` adds
  `("I-007", "GeneratedDrainFenceClear")`; the manifest sets I-007's alloy lane
  `required` with that check_id.
- `scripts/check-verification-full.sh` adds `drain_with_active_lease_invalid` to the Alloy
  negative-control set; coverage/docs/inventory updated (derived, not hand-asserted).

## Non-vacuity proof (anti-theatre)

The new assertion is genuinely discriminating, verified in both directions on the
freshly generated `.als` (AlloyRunner, SAT4J):

- `drain_with_active_lease_invalid` (active exclusive lease on `environment:staging`
  + a drain fence acquired over that same scope): `GeneratedDrainFenceClear`
  resolves **satisfiable=true, holds=false** (Alloy finds the counterexample) →
  whole-model **status=fail (refuted)**; the other checks
  (`GeneratedTraceSatisfiesCore`/`…RuntimePredicatesHaveBarriers`/`…MergeableDefaultEmpty`)
  still hold. The drain assertion alone is what fails it — that delta is the proof.
- The positive `release_promote_success` scenario still passes (**status=pass**);
  `GeneratedDrainFenceClear` is **present** in its `.als` (so I-007 is counted by
  `present_obligations`) and holds vacuously over `DrainFence = none`.
- `causlane_core.als` is **not** touched, so `ValidTraceExists` and the
  I-001/002/003/008 core checks are unaffected.
