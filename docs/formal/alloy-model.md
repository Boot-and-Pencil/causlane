# Alloy model (FM-001)

The Alloy lane answers: *"can a bad finite world exist that the kernel guards
accidentally allow?"* It is the relational / counterexample lane.

## Authority

The hand-written core (`verification/formal-full/alloy/core/causlane_core.als`) is a **generic
reusable checker; authoritative only with generated bundle/scenario facts**. It
is not authority on its own — the compiled bundle + generated facts + receipts +
stale-check are (ADR-0014).

## Files

- `verification/formal-full/alloy/core/causlane_core.als` — generic Event/Action/EventKind model +
  `pred Enforced` guards + the named checks below.
- `verification/formal-full/alloy/generated/*.als` — bundle/scenario facts generated by
  `causlane-codegen` (each carries a provenance header and is stale-checked).
- `verification/formal-full/alloy/checks/unconstrained_counterexample.als` — vacuity/negative
  control for the smoke gate.
- `verification/formal-full/tools/AlloyRunner.java` — headless runner (SAT4J); emits JSON `status`.

## Checks (named per invariant)

| check | invariant | status |
|---|---|---|
| `I_001_ExecutionRequiresBarrier` | I-001 | real, runs in smoke |
| `I_002_ObservedRequiresExecution` | I-002 | real, runs in smoke |
| `I_003_ProjectionRequiresAnchor` | I-003 | real, runs in smoke |
| (generated) `GeneratedRuntimePredicatesHaveBarriers` | I-001 (bundle) | real |
| (generated) `GeneratedMergeableDefaultEmpty` | I-006 (bundle) | real |
| `run ValidTraceExists` | vacuity | proves the guards are non-vacuous |

The generic core directly checks I-001/I-002/I-003 and generated bundle facts
add bundle-level checks such as route/barrier and merge-default consistency. The
remaining invariants are carried by generated obligations and cross-lane
coverage: P monitors, Kani harnesses and replay-backed negative controls. Verus
is an optional proof/spec facet and is not a base/rust gate.
`docs/formal-exceptions.md` records no open P0 exceptions.

## Discrimination (not vacuous)

`verification/formal-full/smoke.sh` runs the core (expect `pass`), the generated success scenario
(expect `pass`) and the generated negatives `execution_without_barrier_invalid`
and `conflicting_leases_invalid` (expect `fail`). It exits 0 only if the
positives hold AND the negatives are refuted — proving the lane is real and
discriminating (R-002).
