# M11.5 Dependency Hygiene Baseline

Date: 2026-06-29

## Summary

`cargo-deny` advisory, license, ban and source checks pass for the workspace.
M11.5 turns duplicate-version handling from a broad warning into a strict deny
gate with exact, reviewed skip entries for the currently known transitive
duplicate lines.

This is not an advisory suppression and does not widen license or source
exceptions. Future duplicate versions must either be resolved or explicitly
reviewed in `deny.toml`.

## Follow-up Policy

- Remove exact skip entries when upstream dependency lines converge.
- Do not add broad `skip-tree` entries for duplicate-version hygiene.
- Do not add advisory ignores or broad license/source exceptions to make a
  release gate pass.
- Treat any new duplicate-version failure as a release-hygiene review item with
  dated evidence.
